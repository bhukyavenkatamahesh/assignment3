\section*{Introduction}

In this assignment, I worked on hand gesture recognition using the Soli
millimeter wave radar dataset. The task had two goals:
\begin{itemize}
\item Improve recognition for the four fine-grained gestures
(pinch index, pinch pinky, finger slider, finger rub).
\item Improve cross-subject generalization using two-fold subject
cross-validation.
\end{itemize}

The model was trained from scratch with adversarial training and a GAN.
No pretrained networks were used. All loss functions, the Gradient
Reversal Layer and the PGD attack were implemented manually.

I called my final model \textbf{AdvSoli}. It is a small custom encoder
on top of which two adversarial signals are applied: a conditional GAN
(cGAN) for fine-grained augmentation, and a Projected Gradient Descent
(PGD) attack on the input. I also tried a third signal (DANN with
Gradient Reversal Layer), but it did not help in this small dataset
setting, so I dropped it from the final model.

\section{Part 1: Approach}

\subsection{Why Adversarial Training and GAN}

The Soli range-Doppler maps are very sparse $32\times32$ images.
Standard CNN methods from the visual domain do not work well because:
\begin{itemize}
\item Most pixels are background; the gesture appears as one or two
small blobs.
\item The fine-grained gestures look almost identical in the radar view.
\item The radar signature changes a lot between people.
\end{itemize}

To handle these, I used three ideas:
\begin{itemize}
\item \textbf{cGAN augmentation}: synthesize extra clips for the four
hard classes so the classifier has more training data for them.
\item \textbf{PGD adversarial training}: train the encoder to be
robust against small input perturbations. This acts as a strong
regularizer and helps generalization.
\item \textbf{DANN (tried but dropped)}: a subject discriminator with
Gradient Reversal Layer to make the encoder ignore subject identity.
This was the natural idea for cross-subject generalization but did not
work in my setting.
\end{itemize}

\subsection{Encoder + Classifier}

I built a small custom encoder from scratch with the following structure:
\begin{itemize}
\item \textbf{Per-frame 2D CNN}: 3 convolutional blocks (16, 32, 64
channels), each with $3\times3$ kernels, batch normalization, leaky
ReLU and $2\times2$ max pool. Spatial size goes from $32 \to 4$.
\item \textbf{Bidirectional GRU}: takes the flattened per-frame
features (1024-dim) and outputs hidden states with 128 units in each
direction.
\item \textbf{Temporal mean pool} of the GRU outputs.
\item \textbf{Linear projection} to a 128-dim feature.
\item \textbf{Classifier head}: small MLP (128 $\to$ 64 $\to$ 11) with
dropout 0.3.
\end{itemize}

I picked 2D-CNN per frame + GRU instead of full 3D CNN because the
RDM is sparse, and a 2D CNN can isolate the small motion blobs per
frame, then the GRU ties their trajectory across time.

\subsection{Conditional GAN}

The generator and discriminator are both small 3D ConvNets:
\begin{itemize}
\item \textbf{Generator}: takes a 64-dim Gaussian noise vector and a
class embedding, projects to a $4\times4\times4$ feature volume, then
3 ConvTranspose3d blocks scale up to $32\times32\times32$, followed by
a $1\times1\times1$ projection to 4 channels and a sigmoid.
\item \textbf{Discriminator}: 3D CNN on the clip, concatenated with
the class embedding, followed by a linear head (real / fake score).
\end{itemize}

The GAN is conditional on class label and only generates the four hard
classes. I used a non-saturating GAN loss with one-sided label
smoothing (real $= 0.9$). I update the discriminator every 2 batches
to keep the GAN balanced, since otherwise the discriminator wins very
fast on this small dataset.

After a short warmup of 5 epochs, the generated fakes are fed through
the encoder and classifier as additional training samples for the
hard classes (with the generator detached, so the GAN loss does not
flow back from the classifier).

\subsection{PGD Adversarial Training}

For each batch, I generate an $\ell_\infty$ adversarial example:
\[
x^* = \Pi_{\|x^* - x\|_\infty \le \epsilon}\Big(x^* + \alpha \cdot
\mathrm{sign}(\nabla_{x^*} L(C(E(x^*)), y))\Big)
\]
with $\epsilon = 0.03$, $\alpha = 0.01$, and 2 iterations starting from
a random point inside the $\epsilon$-ball. The classifier is trained on
both clean and adversarial inputs.

\subsection{DANN Subject Discriminator (Negative Result)}

I also tried the DANN approach: a small subject head behind a Gradient
Reversal Layer. The GRL is identity in the forward pass and multiplies
the gradient by $-\alpha$ in the backward pass. The schedule is the
original DANN one,
\[
\alpha(p) = \frac{2}{1+\exp(-\gamma p)} - 1, \quad \gamma = 10.
\]
I expected this to improve cross-subject generalization. It did not.
I report this as a negative finding in Section 3.

\subsection{Combined Loss}

The final AdvSoli loss is:
\[
\mathcal{L} = \mathcal{L}_{\text{cls}} + \lambda_p \mathcal{L}_{\text{pgd}}
+ \lambda_g \mathcal{L}_{\text{aug}}
\]
with $\lambda_p = 0.1$, $\lambda_g = 0.2$. The GAN itself (G and D)
has its own optimizer.

\section{Part 2: Experiments}

\subsection{Dataset}

I used the public Soli range-Doppler dataset:
\begin{itemize}
\item 11 gestures.
\item 10 subjects (sessions 2, 3, 5, 6, 8, 9, 10, 11, 12, 13).
\item 25 instances per (gesture, subject), total 2,750 clips.
\item Each clip is a sequence of $32\times32$ RDM frames across 4
radar channels (HDF5 format).
\end{itemize}

Each clip was padded or center-cropped to 32 frames and min-max
normalized to $[0, 1]$.

For two-fold cross-validation:
\begin{itemize}
\item \textbf{Fold 0}: train on subjects $\{2, 3, 5, 6, 8\}$, test on
$\{9, 10, 11, 12, 13\}$.
\item \textbf{Fold 1}: swap.
\end{itemize}

\subsection{Training Setup}

\begin{itemize}
\item Optimizer: Adam, learning rate $3 \times 10^{-4}$, weight decay
$10^{-4}$.
\item Batch size: 16.
\item Epochs: 30.
\item Total parameters: 1.6M.
\item Hardware: MacBook Air (Apple Silicon, MPS backend).
\item Per-fold training time: 12 - 16 minutes.
\end{itemize}

\section{Part 3: Discussion}

\subsection{What Worked}

\begin{itemize}
\item \textbf{PGD adversarial training} was the single most useful
component. Overall accuracy improved by 1.9 percentage points and
robustness under a PGD attack went from 2.4\% to 27.3\%.
\item \textbf{cGAN augmentation} was the most useful for fine-grained
accuracy (+1.3 points). The pinch classes especially benefited.
\item The combined \textbf{AdvSoli (cGAN + PGD)} model preserves
clean accuracy and gives 21.9\% PGD-robust accuracy, which is around
9 times better than the backbone.
\end{itemize}

\subsection{What Did Not Work: DANN}

DANN consistently hurt accuracy in my experiments. I think the reason
is:
\begin{itemize}
\item Each fold has only 5 training subjects.
\item The subject discriminator has very little information to learn
from. Its loss stays close to $\log(10) \approx 2.3$, near random.
\item The GRL feeds mostly noise back into the encoder gradients.
\end{itemize}

DANN typically needs many source domains to work well. In my setting
the assumption is violated.

\subsection{Why the Components Do Not Compose Cleanly}

Adding cGAN on top of a PGD-trained model brings overall accuracy back
to baseline. My interpretation:
\begin{itemize}
\item PGD already pushes the encoder towards input-robust features.
\item Synthetic GAN clips look like another input perturbation rather
than a useful new mode.
\item The cGAN biases the encoder slightly towards typical pinch
patterns, hurting other fine-grained classes.
\end{itemize}

I still ship cGAN + PGD as the AdvSoli model because:
\begin{itemize}
\item The assignment requires both adversarial training and GANs.
\item Robustness improves 9 times with no clean-accuracy cost.
\item The cGAN noticeably helps the two pinch classes.
\end{itemize}

\section*{Conclusion}

In this assignment, I built AdvSoli, a small custom adversarial stack
for the Soli radar gesture task. It uses a 2D-CNN + Bi-GRU encoder
trained jointly with:
\begin{itemize}
\item a conditional GAN that synthesizes extra clips for the four
fine-grained classes,
\item and a PGD adversarial perturbation on the input.
\end{itemize}

On two-fold cross-subject CV, the model maintains the strong backbone
clean accuracy (86.6\% vs 86.5\%) and improves adversarial robustness
by roughly 9 times.

I also reported a negative finding on DANN: in a setting with only
five training subjects per fold, the subject discriminator does not
help and actually hurts performance. The honest ablation table makes
clear which adversarial signal contributes to which axis (cGAN for
fine-grained, PGD for cross-subject and robustness).
